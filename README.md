# W0PIUM — Социальная сеть

## 1. Общее описание

W0PIUM — закрытая социальная сеть с регистрацией только по инвайт-кодам. Пространство для артистов и близкого круга: лента постов, истории (дропы), личные сообщения, облачное хранилище и система уведомлений — всё в одном месте.

**Для кого:** приватное сообщество, доступ через инвайт.  
**Основные цели:** общение, публикация контента, обмен файлами.

## 3. Реализованный функционал

### Авторизация и профили

- Регистрация по инвайт-коду (режим `INVITE_ONLY`)
- Верификация email — 6-значный код на почту (Resend API)
- Вход / выход, сброс и смена пароля
- Управление сессиями: список активных устройств, выход из всех сразу
- Редактирование профиля: имя, аватар, обложка, био
- Приватный аккаунт — подписки требуют одобрения
- Подписки / подписчики; рекомендации людей для подписки
- Блокировка и мьют пользователей
- Значки верификации (заявка → одобрение/отклонение админом)
- Экспорт своих данных (посты, профиль)

### Лента и публикации

- Лента подписок (`/feed`) и глобальный обзор (`/discover`)
- Посты: текст, изображение (авто-сжатие, поддержка HEIC/HEIF), embed SoundCloud
- Опросы в постах (несколько вариантов, отображение результатов)
- Отложенная публикация (планировщик с datetime)
- Лайки, реакции эмодзи (🖤 💀 🥀 👁️ 🌹 🔮), комментарии
- Репосты (с текстом или без)
- Закладки, архивирование, редактирование, удаление, закреп на профиле
- Список тех, кто лайкнул
- Предпросмотр ссылок (link preview по OG-тегам)
- Хэштеги с отдельными страницами

### Дропы (Drops)

- Исчезающий контент — пропадает через 24 часа
- Текст + изображение
- Счётчик просмотров (IntersectionObserver)
- Лента дропов отдельная от основной

### Чаты и сообщения

- Личные чаты 1-на-1 и групповые (аватар, название группы)
- Система запросов на переписку (DM request / принять / отклонить)
- Текстовые сообщения, вложения (изображения, файлы)
- **Голосовые сообщения** — запись через микрофон, предпросмотр перед отправкой, прослушивание с регулировкой скорости, скачивание
- Реакции эмодзи на сообщения
- Редактирование и удаление сообщений
- Ответ на сообщение (reply), пересылка в другой чат (forward)
- Закреп сообщения в чате
- Тайпинг-индикатор
- Галерея медиафайлов чата (изображения / файлы / голосовые)
- Поиск по переписке
- Мьют чата
- Экспорт переписки
- Прочтение — двойная галочка ✓✓
- Real-time через Server-Sent Events (SSE)

### Поиск

- Полнотекстовый поиск: посты, пользователи, хэштеги
- Поиск по сообщениям
- Страница `/artists` — каталог всех пользователей

### Диск (Disk)

- Облачное файловое хранилище
- Папки: создание, переименование, перемещение, удаление
- Загрузка файлов (до 4 ГБ), breadcrumb-навигация
- Пакетное скачивание выбранных файлов в ZIP
- Публикация файла по публичной ссылке (`/pub/:token`)
- Статистика использования хранилища

### Уведомления

- Центр уведомлений (лайки, комментарии, упоминания, подписки, верификация)
- Web Push-уведомления (VAPID) — приходят при закрытой вкладке
- Счётчик непрочитанных в навигации

### Администрирование и модерация

- Статистика платформы
- Управление пользователями: бан, удаление, повышение до админа
- Модерация: жалобы (resolve/reject), дропы, посты
- Одобрение / отклонение заявок на верификацию
- Просмотр всех инвайт-кодов

### PWA

- Устанавливается как приложение (Add to Home Screen)
- Service Worker: кеш-фёрст для статики, network-only для API
- Автообновление при деплое новой версии

## 4. Технический стек

| Слой | Технология |
|---|---|
| Runtime | Node.js 20 + Express 4 |
| База данных | better-sqlite3 (SQLite, файловая) |
| Frontend | Vanilla JS SPA — без фреймворков |
| CSS | Кастомные CSS-переменные + Tailwind utilities (Vite/PostCSS) |
| Realtime | Server-Sent Events (`/api/events`) |
| Email | Resend API (верификация, сброс пароля) |
| Push | Web Push / VAPID |
| Файлы | Multer → локальный диск в `DATA_DIR/` |
| Безопасность | Helmet, bcryptjs (cost 10), CSRF-токены, express-rate-limit |
| Хостинг | Docker на Synology NAS, доступ через Cloudflare Tunnel |
| Деплой | `docker compose up --build -d` |

## 5. Как запустить проект локально

```bash
# 1. Установить зависимости
npm install

# 2. Создать .env из примера и заполнить значения
cp .env.example .env

# 3. Запустить сервер (порт 3000)
npm start

# 4. (Опционально) Vite dev-сервер для CSS hot-reload (порт 5173)
npm run dev
```

Открыть: **http://localhost:3000**

> Без Vite тоже работает полностью — Express отдаёт `public/` как статику.

Переменные окружения (`DATA_DIR`, `RESEND_API_KEY`, `VAPID_*` и др.) — см. раздел 7.

## 7. Особенности и известные ограничения

**Переменные окружения** (скопировать `.env.example` → `.env`):

| Переменная | Описание |
|---|---|
| `PORT` | Порт сервера (по умолчанию `3000`) |
| `NODE_ENV` | `development` / `production` |
| `DATA_DIR` | Путь к директории с данными (БД, файлы) |
| `INVITE_ONLY` | `1` — только по инвайту, `0` — открытая регистрация |
| `MASTER_CODE` | Мастер-инвайт-код (всегда работает) |
| `RESEND_API_KEY` | API-ключ Resend для отправки писем |
| `EMAIL_FROM` | Адрес отправителя (`onboarding@resend.dev` по умолчанию) |
| `ENCRYPTION_KEY` | Ключ шифрования (64 hex-символа) |
| `VAPID_PUBLIC` | Публичный VAPID-ключ для Web Push |
| `VAPID_PRIVATE` | Приватный VAPID-ключ для Web Push |

Сгенерировать VAPID-ключи:
```bash
node -e "const wp = require('web-push'); const k = wp.generateVAPIDKeys(); console.log('PUBLIC:', k.publicKey, '\nPRIVATE:', k.privateKey);"
```

**Известные ограничения:**

- **Тема:** тёмная (по умолчанию) + светлая. Переключается через навигацию, сохраняется в `localStorage`.
- **Масштабирование:** SQLite — один файл, один процесс. При большой нагрузке нужна миграция на PostgreSQL.
- **Email:** без своего домена в Resend — лимит 100 писем/день (бесплатный план).
- **Push-уведомления:** требуют HTTPS и заполненных `VAPID_*` ключей.
- **Голосовые сообщения:** зависят от `MediaRecorder API` — не работают в старых Safari.
- **Лимиты файлов:** изображения — 5 МБ, прочие файлы — 4 ГБ.
- **Hub / федерация:** страница присутствует, распределённая часть в разработке.
